# 04-ladenheim / 03 — Stage D: reproduce the 62 realizability-set equivalence classes

Status: `todo`

## Revision note (2026-07-10)

This task previously read "Investigate equivalence-class grouping as a
separate layer from raw catalogue enumeration," without stating what
relation defines the 62 classes or how they differ from other groupings of
the same 108 networks. See `docs/ladenheim_benchmark.md` for the full
reassessment.

Source: A. Morelli and M. C. Smith, *Passive Network Synthesis: An Approach
to Classification*, SIAM, 2019, Ch. 6; Thm. 7.4.

## Goal

Classify the 108-network canonical set (Stage C, `02-108.md`) into
equivalence classes defined by equality of **realizability set** — the set
of positive-real driving-point immittances a network can realize as its
positive component values vary. This is a behaviour/realizability-based
relation, not a further pass of graph or topological canonicalization, and
it very likely requires algebraic network analysis (symbolic driving-point
impedance derivation and comparison), not just graph-level reasoning.

This stage depends on Stage C (`02-108.md`) being complete: there must be an
independently-reproduced 108-network canonical set to classify.

## Three distinct groupings of the same 108 networks — do not conflate

Morelli & Smith describe three different groupings of the 108-network
canonical set. They partition or group the same 108 networks by different
criteria, and none is a substitute for another:

- **62 realizability-set equivalence classes** (this task's target): grouped
  by equality of realizability set as component values vary.
- **35 group-action orbits**: a separate grouping, not the target of this
  task.
- **24 subfamilies**: a separate grouping, not the target of this task.

Documentation and code produced by this task must always name which of the
three (62, 35, or 24) is meant, and must not use "equivalence classes" as an
unqualified stand-in for any of them.

## Tasks

- Confirm the precise definition of "realizability set" used by the source
  (which parameters vary, whether it is the set of achievable driving-point
  impedances, admittances, or both) before implementing anything.
- Investigate what machinery is needed to compare realizability sets: this
  is expected to require deriving each network's driving-point immittance as
  a function of its component values (symbolically or parametrically) and
  testing for equality of the resulting achievable sets, not merely
  comparing topologies.
- Do not pre-implement a topology-only proxy for this relation; if a
  topology-only necessary condition is found useful as a first filter, label
  it explicitly as a filter, not as the classification itself.
- Reproduce the 35-orbit and 24-subfamily groupings separately if attempted,
  clearly labelled as distinct from the 62 classes, so all three can be
  cross-checked against each other and against the source without being
  merged into one number.

## Done means

- The 62-class grouping is reproduced (or its required machinery is
  documented as a distinct, scoped follow-up) using a realizability-set
  relation, not a topological canonicalization.
- The 35-orbit and 24-subfamily groupings, if reproduced, are kept visibly
  distinct from the 62 classes throughout.
- Counts, enumeration results, catalogues, equivalence classes, generator
  sets, and full immittance identity are not conflated.
