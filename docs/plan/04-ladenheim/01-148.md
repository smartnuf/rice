# 04-ladenheim / 01 — Reproduce 148 essentially distinct Ladenheim networks

Status: `todo`

## Goal

Define, implement, and validate the historical structural enumeration whose
acceptance target is the **148 essentially distinct networks** reported by
Morelli and Smith.

## Correct contract to define before implementation

The 148 are not raw labelled candidates and are not ordinary graph-isomorphism
classes. The historical construction starts from connected two-terminal basic
graphs with at most five branches, labels branches by primitive `R`, `L`, or `C`
elements with at most two reactive elements, identifies deformation,
separation, and series interchange via graph 2-isomorphism, and excludes
trivially simplifiable same-kind series or parallel connections.

Initial scope:

```text
total elements <= 5
L + C <= 2
```

There is no initial `R <= 3` restriction. Eight four-resistor/one-reactive
networks are part of the 148 and are removed only in the later 108-network
catalogue stage.

## Required plan

1. Write the historical 148 model contract, including terminal orientation,
   2-isomorphism, same-kind simplification exclusion, and any distinguished
   port/source-edge convention.
2. Implement a dedicated Ladenheim structural enumeration/signature separate
   from the current RICE reduced signature unless a later proof shows the two
   contracts coincide.
3. Report diagnostic tables by total element count and component composition,
   not only the final integer 148.
4. Add tests/fixtures that make mismatches inspectable.

## Current RICE diagnostic, not the target

The current RICE reduced signature gives 140 signatures for the old planned
slice `R <= 3`, `L+C <= 2`, support edges `<= 5`, reproduced at commit `338ddec`
with:

```bash
.venv/bin/python -m rice reduced --max-r 3 --max-reactive 2 --max-edges 5 --format json
```

This verifies the project-specific RICE reduced model for that slice. It does
not reproduce the historical 148 because the historical scope and structural
relation differ.
