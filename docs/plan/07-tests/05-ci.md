# 07-tests / 05 — CI-friendly validation commands

Status: `prog`

## Goal

Make validation clear for humans, Codex, and future CI.

## Required commands

- Fast setup.
- Fast tests.
- Optional slow enumeration tests.
- Full local check.

## Done means

- `make test` and `scripts/test.*` run the fast suite.
- Slow tests can be enabled explicitly.
- Command output shows the Python executable in use.

## Progress notes

- The Makefile provides CI-friendly local commands: `make setup`, `make test`,
  `make supports`, legacy count checks, and aggregate `make check`, all using
  `.venv/bin/python`.
- `.codex/setup.sh` and `.codex/maintenance.sh` exist for Codex Cloud setup and
  cached-environment maintenance.
- Bash/Linux/WSL validation scripts now provide the non-Make path:
  `./scripts/setup.sh`, `./scripts/test.sh`, `./scripts/lint.sh`,
  `./scripts/check.sh`, and `./scripts/clean.sh`.
- `make setup`, `make test`, `make lint`, `make check`, and `make clean` now
  delegate to the Bash scripts.
- Native PowerShell scripts and explicit slow-test selection are still missing,
  so this remains `prog`.

## Near-term next steps

1. Add native Windows PowerShell wrappers before marking this item `done`.
2. Decide how explicit slow-test selection should be exposed in CI-oriented
   commands.
